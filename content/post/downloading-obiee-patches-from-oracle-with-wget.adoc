Last week saw the release of link:https://www.rittmanmead.com/blog/2012/07/obiee-11-1-1-6-2-bp1-new-features-oracle-bi-mobile-hd-and-patching-upgrade-considerations/[OBI 11.1.1.6.2 BP1], and with it, some eight patches to download (seven OBIEE, plus one JDeveloper). A very useful option when downloading the patches, particularly if you are working on Linux servers with no GUI, is to download the patches with link:http://www.gnu.org/software/wget/?ref=rittmanmead.com[wget]. This can also be good if your company has download policies that necessitate a third party downloading any external files.


The option for downloading via wget from Oracle Support is not immediately obvious, so here is how to do it, using the example of the patchset for OBI 11.1.1.6.2 BP1 :


First, you need to get a list of all the patches you want to download. Log into My Oracle Support (link:https://support.oracle.com/?ref=rittmanmead.com[Flash version] or link:https://supporthtml.oracle.com/?ref=rittmanmead.com[HTML version]), and click on *Patches & Updates*


image::/images/2012/07/061285MOS.png[MOS]


Click on *Product or Family (Advanced)*and enter the following criteria to find the patches for OBIEE 11.1.1.6.2 BP1:


* *Product*: Oracle Business Intelligence
* *Release*: OBI EE 11.1.1.6.2BP1
* *Platform*: Linux x86-64

Also make sure you tick *Include all products in a family*.


image::/images/2012/07/061284Untitled2.png[NewImage]


Click on *Search*, and you should get a list of results including seven for the patchset we are interested in:


image::/images/2012/07/061286Untitled-1.png[NewImage]


Click on the first patch in the list, then press *Shift* and click on the last patch that we want. This will select all seven patches, and display a *Download*button. Click this button. 

image::/images/2012/07/061283Untitled-2.png[NewImage]


In the bottom left of the *File Download*window, you should see a link for *WGET Options*.


*image::/images/2012/07/061282Untitled-3.png[NewImage]*


Click on *WGET Options* and then on the *Download .sh*button in the following dialogue


image::/images/2012/07/061279NewImage4.png[NewImage]


Now open the *wget.sh*file that was downloaded, and locate the lines for *SSO_USERNAME*and *SSO_PASSWORD*. Edit these with your My Oracle Support username and password - the username will default to the user that created the wget.sh file.


[source,shell]
----

#!/bin/sh

Generated 7/2/12 7:46 AM
Start of user configurable variables

LANG=C
export LANG
SSO username and password
SSO_USERNAME=larry@oracle.com
SSO_PASSWORD=password
----


Set the script as executable:


[source]
----
chmod u+x wget.sh
----
and then execute it:

[source]
----
./wget.sh
----

The script will write a corresponding wget.log with the output of the session, and if all is well you should see the patch files written to the current directory. If the files don't download, check the .log file for the cause. 


== Footnote


Whilst My Oracle Support only lists seven patches for 11.1.1.6.2 BP1 when you search using the method above, per link:https://www.rittmanmead.com/blog/2012/07/obiee-11-1-1-6-2-bp1-new-features-oracle-bi-mobile-hd-and-patching-upgrade-considerations/[Mark's article], you also need the patch for JDeveloper, 13952743.
